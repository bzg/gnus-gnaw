#+title: gnus-bone.el --- Highlight BARK reports in Gnus

[[https://intver.org/][https://img.shields.io/badge/versioning-intver.org-blue.svg?style=for-the-badge]]

=gnus-bone= highlights Gnus summary lines whose =Message-Id= matches an
open [[https://codeberg.org/bzg/bark][BARK]] report. It reads the same =reports.json= sources as [[https://codeberg.org/bzg/bone][bone]].

* Requirements

- The [[https://codeberg.org/bzg/bone.el][bone.el]] library, which provides the shared data layer
  (configuration, report cache and =state.edn= handling).  Once
  =gnus-bone= is on MELPA this dependency will be pulled in
  automatically; until then, see /Installation/ below.
- =~/.config/bone/config.edn= (an EDN map with =:my-addresses= and
  =:sources= keys, shared with bone)
- =gnus-bone= supports =reports.json= bark-format =0.9.3=.

* Installation

=gnus-bone= is not on MELPA yet.  On Emacs 30 or later, install it from
git with the built-in =package-vc=.  Install [[https://codeberg.org/bzg/bone.el][bone.el]] *first*, since it
is its dependency and is not in any package archive:

#+begin_src emacs-lisp
(use-package bone
  :vc (:url "https://codeberg.org/bzg/bone.el" :rev :newest))

(use-package gnus-bone
  :vc (:url "https://codeberg.org/bzg/gnus-bone" :rev :newest))
#+end_src

Or interactively, in order:

: M-x package-vc-install RET https://codeberg.org/bzg/bone.el RET
: M-x package-vc-install RET https://codeberg.org/bzg/gnus-bone RET

Update later with =M-x package-vc-upgrade=.

=gnus-bone= does not require the [[https://codeberg.org/bzg/bone][bone]] CLI: it reads =config.edn= and
=reports.json= files directly.  You can use it without installing the
=bone= binary, as long as the configuration file and the report sources
it references are available.

* Commands

- =M-x gnus-bone= — Apply overlays to the current summary buffer.
- =M-x gnus-bone-highlight= — Highlight without limiting the summary.
- =M-x gnus-bone-topic= — Filter reports by selected topic.
- =M-x gnus-bone-clear= — Remove all =gnus-bone= overlays.

** Local marks

The following commands toggle bone's local marks (stored in
=~/.config/bone/state.edn= and shared with the [[https://codeberg.org/bzg/bone][bone]] CLI).

- =M-x gnus-bone-mark-sticky= — toggle the sticky mark (keep the report visible).
- =M-x gnus-bone-mark-skip= — toggle the skip mark (hide the report).

The annotation gains a leading mark column: =*= for sticky, =_= for
skip, space otherwise.
